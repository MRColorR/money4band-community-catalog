# money4band-community-catalog

Community-maintained catalog of apps and addons for
[Money4Band (M4B)](https://github.com/MRColorR/money4band) — entries that
are **external, experimental, or not officially supported** by M4B core.

This repo is one of three M4B catalog sources:

| Repo | Scope |
|------|-------|
| [`MRColorR/money4band`](https://github.com/MRColorR/money4band) | Core CLI + ToS-compliant supported apps (built-in) |
| **`MRColorR/money4band-community-catalog`** (this repo) | Community apps + auto-claim addons (opt-in remote source) |
| [`MRColorR/openpassivehub-catalog`](https://github.com/MRColorR/openpassivehub-catalog) | Broad public ecosystem directory (informational) |

## Layout

All three repos share an identical per-file layout so entries can move
between them by plain file rename — no schema translation:

```
catalog/
├── supported/          # Community apps considered stable
│   └── <NAME>.json
├── newcomers/          # Experimental / 6-month evaluation
│   └── <NAME>.json
└── deprecated/         # Retired / broken — kept for history
    └── <NAME>.json
addons/                 # Sidecar containers (e.g. auto-claim pots)
└── <NAME>.json
manifest.json           # Aggregated wire format, regenerated by CI
scripts/
└── build_manifest.py   # Shared with core (keep in sync)
```

Each `<NAME>.json` follows the M4B app-bundle schema documented in
[`money4band/docs/app-management.md`](https://github.com/MRColorR/money4band/blob/main/docs/app-management.md#app-bundle-schema).

## Using this catalog from M4B

Add a `remote` source to your `config/m4b-config.json → catalog.catalog_sources`:

```jsonc
{
  "type": "remote",
  "url":  "https://raw.githubusercontent.com/MRColorR/money4band-community-catalog/main/manifest.json",
  "default_source": "community",
  "cache_path": "catalog/community",
  "enabled": true,
  "timeout": 5.0
}
```

M4B fetches `manifest.json` with ETag caching, writes per-file bundles
into `config/catalog/community/`, and merges them into the app list.
When the network is unavailable, the last cached copy is used.
