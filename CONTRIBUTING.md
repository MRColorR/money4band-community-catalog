# Contributing to money4band-community-catalog

Thanks for helping curate M4B's community catalog!

## TL;DR

1. Add or edit one file under `catalog/<category>/<NAME>.json` (or
   `addons/<NAME>.json`) per pull request.
2. Follow the M4B app-bundle schema documented in
   [`money4band/docs/app-management.md`](https://github.com/MRColorR/money4band/blob/main/docs/app-management.md#app-bundle-schema).
3. Filename stem = `name` field (uppercase, `[A-Z][A-Z0-9_]+`).
4. `manifest.json` is regenerated by CI — do **not** edit by hand.

## Categories

| Directory | Meaning |
|-----------|---------|
| `catalog/newcomers/` | New or experimental apps; standard landing spot |
| `catalog/supported/` | Promoted after 6 months of stable community use |
| `catalog/deprecated/` | Dead/retired — kept for history, with `additional_details` explaining why |
| `addons/` | Sidecar containers (e.g. auto-claim pots) with a `parent_id` |

## Adding an entry

```bash
cp catalog/newcomers/TEMPLATE.json catalog/newcomers/MYAPP.json  # if template exists
# ...edit fields...
python scripts/build_manifest.py --check   # verify manifest is up-to-date (CI runs this)
```

Then open a PR. CI validates JSON structure and rebuilds `manifest.json`.

## Referrals

Add your referral link to the entry's `referrals[]` pool:

```jsonc
"referrals": [
  { "url": "https://r.example/?ref=YOURCODE", "contributor": "YourHandle",
    "added": "2026-04-24", "verified": false }
]
```

Links are picked by weighted rotation driven by
`scripts/compute_weights.py` in M4B core — the more you contribute
across any of the three repos, the more weight your link carries.

## What belongs here vs. core

- **core** — only ToS-compliant, officially supported apps.
- **community-catalog (here)** — apps the core team can't ship (ToS
  grey area, requires manual install, auto-claim sidecars, etc.).
- **openpassivehub-catalog** — everything else, including non-M4B tools
  and external services.

If unsure, open an issue first.
